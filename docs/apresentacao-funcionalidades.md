# Proposta inicial de sistema de fluxo de caixa

## Objetivo

A ideia é desenvolver um sistema simples, online e prático para ajudar uma loja/franquia a controlar melhor o fluxo de caixa.

Nesta primeira etapa, o foco não é criar um sistema complicado. O objetivo é transformar o controle que hoje seria feito em PDF, planilha ou anotações em um sistema integrado, mais organizado e fácil de consultar.

## O que o sistema vai ajudar a responder

Com o sistema, será possível saber rapidamente:

- quanto entrou no dia;
- quanto saiu no dia;
- quanto sobrou de lucro estimado no dia;
- quanto entrou no mês;
- quanto saiu no mês;
- quanto sobrou de lucro estimado no mês;
- quais lançamentos formam esses valores.

## Primeira versão do sistema

A primeira versão será focada no essencial:

```txt
Entradas + Saídas + Lucro estimado diário + Lucro estimado mensal
```

A proposta é começar com algo direto, útil e fácil de usar.

## Funcionalidades pensadas para a primeira etapa

### 1. Acesso online

O sistema poderá ser acessado pelo computador, notebook, tablet ou celular.

Assim, o dono poderá acompanhar o financeiro mesmo quando não estiver fisicamente na loja.

### 2. Cadastro de entradas

Será possível registrar tudo que entrou no caixa da empresa.

Exemplos:

- venda de veículo elétrico;
- venda de peça;
- venda de acessório;
- serviço de assistência;
- Pix recebido;
- dinheiro recebido;
- transferência recebida;
- venda no cartão.

Cada entrada poderá ter:

- descrição;
- valor;
- data;
- categoria;
- forma de recebimento;
- observação, se necessário.

### 3. Cadastro de saídas

Será possível registrar tudo que saiu do caixa da empresa.

Exemplos:

- compra de veículos;
- compra de peças;
- frete;
- aluguel;
- energia;
- internet;
- salário;
- comissão;
- marketing;
- contador;
- impostos;
- taxas;
- manutenção da loja;
- outras despesas.

Cada saída poderá ter:

- descrição;
- valor;
- data;
- categoria;
- forma de pagamento;
- observação, se necessário.

### 4. Resumo diário

O sistema mostrará o resultado de cada dia.

Exemplo:

```txt
Entradas do dia: R$ 8.350,00
Saídas do dia: R$ 1.650,00
Lucro estimado do dia: R$ 6.700,00
```

Isso ajuda a entender como foi o movimento financeiro do dia.

### 5. Resumo mensal

O sistema também mostrará o resultado do mês.

Exemplo:

```txt
Entradas do mês: R$ 45.000,00
Saídas do mês: R$ 31.500,00
Lucro estimado do mês: R$ 13.500,00
```

Isso ajuda a acompanhar se a loja está tendo um mês positivo ou se as despesas estão muito altas.

### 6. Organização por categorias

As entradas e saídas poderão ser organizadas por tipo.

Exemplos de entradas:

- venda de veículo;
- venda de peça;
- serviço;
- acessórios;
- outros recebimentos.

Exemplos de saídas:

- compra de veículos;
- peças;
- aluguel;
- energia;
- marketing;
- impostos;
- taxas;
- outras despesas.

Com isso, fica mais fácil entender de onde vem o dinheiro e para onde ele está indo.

### 7. Tela inicial com resumo

A tela inicial poderá mostrar as principais informações de forma rápida:

- entradas do dia;
- saídas do dia;
- lucro estimado do dia;
- entradas do mês;
- saídas do mês;
- lucro estimado do mês.

A ideia é que, ao abrir o sistema, já seja possível entender a situação financeira principal da loja.

## Benefícios esperados

Com essa primeira versão, a empresa cliente poderá:

- reduzir anotações soltas;
- evitar confusão com valores;
- registrar entradas e despesas de forma organizada;
- consultar o resultado do dia;
- consultar o resultado do mês;
- entender melhor se o caixa está positivo;
- acessar informações mesmo fora da loja;
- ter uma base mais segura para tomar decisões.

## Por que começar simples

Começar simples permite criar uma primeira versão mais rápida, útil e fácil de validar.

Depois que a loja usar o sistema no dia a dia, será possível entender quais melhorias fazem mais sentido.

## Possíveis melhorias futuras

Depois da primeira versão, o sistema poderá evoluir para funções como:

- contas a receber;
- contas a pagar;
- controle de vendas parceladas;
- previsão de caixa futuro;
- alertas de contas e despesas;
- relatórios por categoria;
- gráficos;
- exportação em PDF ou Excel;
- controle por usuário;
- integração com outros sistemas.

Essas melhorias podem ser feitas aos poucos, conforme a necessidade real da loja.

## Resumo da proposta

A proposta inicial é criar um sistema financeiro online para responder, de forma simples:

```txt
O que entrou?
O que saiu?
Quanto sobrou?
Como foi o dia?
Como foi o mês?
```

O objetivo é dar mais clareza, organização e controle para o financeiro de um pequeno negócio.
